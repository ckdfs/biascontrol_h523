# Bias Controller Firmware — AI Maintenance Guide

## Project Overview
Optical modulator bias controller firmware for STM32H523CET6.
Uses pilot tone dithering + digital lock-in (Goertzel) for closed-loop bias voltage control.

## Architecture (5 layers, top-down)
```
APP       → app/            State machine, config, startup
CONTROL   → control/        Bias loop, modulator strategies, PID
DSP       → dsp/            Goertzel harmonic extraction, pilot sine generation
DRIVER    → drivers/        ADS131M02 (ADC), DAC8568 (DAC), board GPIO
HAL       → cubemx/         STM32 HAL generated by CubeMX (do NOT edit manually)
              ├── Core/     CubeMX init code (gpio, spi, gpdma, tim, usart, interrupts)
              └── Drivers/  CMSIS + STM32H5xx HAL library
```

## Key Design Patterns
- **Strategy Pattern** for modulator types: each modulator (MZM, DPMZM, etc.)
  implements `modulator_strategy_t` in `control/inc/ctrl_modulator.h`
- **Layered architecture**: upper layers never call HAL directly; they go through drivers
- **DSP layer is pure math**: no hardware dependencies, testable on host

## Naming Conventions
- Files: `<layer>_<module>.c/h` (e.g., `drv_dac8568.c`, `dsp_goertzel.c`, `ctrl_pid.c`)
- Functions: `<module>_<action>()` (e.g., `dac8568_write_channel()`, `goertzel_reset()`)
- Types: `<module>_<name>_t` (e.g., `harmonic_data_t`, `bias_point_t`)
- Defines/enums: `UPPER_SNAKE_CASE`
- All public functions/types declared in corresponding `.h` file

## Hardware Quick Reference (Verified from Netlist 2026-03-21)
- SPI1 → DAC8568 (8-ch, 16-bit): PA5(CLK), PA7(DIN), PB1(SYNC), PB2(LDAC), PB0(CLR)
- SPI2 → ADS131M02 (2-ch, 24-bit): PB13(SCLK), PB15(MOSI), PB14(MISO), PA8(CLKIN/MCO1), PB12(CS), PA11(DRDY), PA12(/SYNC/RESET)
- DAC outputs VOA-VOH → subtractor circuits → VA-VH (-10V~+10V)
- PD → OPA140 TIA → ADC CH0 (AIN0P)
- USART1: PA9(TX), PA10(RX) — debug/tuning interface
- LED: PC13, Button: NRST(SW1)
- GPDMA1: Ch0(SPI1_TX), Ch1(SPI2_TX), Ch2(SPI2_RX), Ch3(USART1_TX), Ch4(USART1_RX)

## Build Commands
```bash
# Cross-compile for target
mkdir -p build && cd build
cmake -DCMAKE_TOOLCHAIN_FILE=../cmake/arm-none-eabi.cmake ..
make -j$(nproc)

# Flash via OpenOCD
openocd -f interface/stlink.cfg -f target/stm32h5x.cfg \
  -c "program biascontrol.elf verify reset exit"

# Host-side unit tests (native compiler)
mkdir -p build-test && cd build-test
cmake -DBUILD_TESTS=ON ..
make && ctest
```

## Adding a New Modulator Type
1. Create `control/src/ctrl_modulator_<name>.c` and `.h`
2. Implement `modulator_strategy_t` interface (compute_error, is_locked, init)
3. Add enum entry in `ctrl_modulator.h` → `modulator_type_t`
4. Register in `modulator_get_strategy()` lookup table
5. No changes needed in ctrl_bias.c or app layer

## Critical Invariants
- Goertzel block size N must ensure integer cycles of pilot frequency (no spectral leakage)
- Harmonic magnitudes must be normalized by DC power before error computation
- DAC output = bias_setpoint + pilot_sample, updated synchronously with ADC
- PI controller must have anti-windup clamping
- DRDY interrupt has highest NVIC priority; control loop runs at lower priority
